## 목표 (Goal)

"디지털 기억 유산"을 주제로 한 싱글 페이지 웹앱 **보따리(Bottari)**를 시안 기반으로 새로 만든다.

서사는 MVP와 동일하게 **선별(Curation) → 비움(Release)** 이다. 사용자는 돌아올 수 없는
먼 길을 떠나는 나그네가 되어, 이름을 짓고, 보자기를 고르고, 소중한 기억(사진·글·지도)을
담는다. 보자기의 수용 개수는 3단계(10/5/3개)로 줄어들며, 1·2단계에서 봇짐을 매면
**담았던 아이템이 그대로 다음 단계로 넘어와** 더 작은 한도에 맞게 덜어내야 한다.
3단계(3개)를 매듭지어야 "비로소, 가벼워졌다!"의 엔딩이 열린다.

완성된 보따리는 두 갈래로 남길 수 있다:
1. **소중한 이에게 맡기기** — 암호를 설정하고, 클라이언트 측에서 암호화된 공유 URL을 전달
2. **보따리 보관함에 맡기기** — 공개 보관함(갤러리)에 남겨 다른 나그네들이 풀어보게 함

MVP가 기능 골격(첨부·암호화·공유)을 검증했다면, 리뉴얼의 초점은
**픽셀아트 비주얼 + 내러티브 연출 + 보관함**으로 경험을 완성하는 것이다.

## 제약 사항 (Constraints)

- **노빌드 바닐라 스택**: 순수 HTML/CSS/JS. npm·번들러·프레임워크 금지. 외부 코드는 CDN 로드만.
- **백엔드 서버 없음**: 상태는 localStorage + URL 파라미터. 보관함만 예외적으로 Supabase REST를
  `fetch()`로 직접 호출하되, **미설정 상태에서도 앱 전체가 동작해야 한다**
  (`js/config.js` 키 2개가 비면 localStorage 모드 자동 폴백 — DESIGN.md §0-1).
- 공유 링크는 Web Crypto API(PBKDF2 + AES-GCM)로 실제 암호화. 암호는 4자 이상.
- 공유 URL 전체 길이 약 2,000자 이내 유지 — 사진은 canvas 단계적 압축(320→48px, 품질 자동 하향),
  초과 시 추가 축소. 기존 배포본의 `image.js` 로직을 계승한다.
- 아이템 한도 10/5/3 고정. 엔딩은 3단계 봇짐 매기에서만 발생. 상위 단계로의 건너뛰기 허용
  (2·3단계 직행 선택 가능), 지난 단계로의 회귀는 불가.
- 재선택 규칙(DESIGN.md §0-4): 이월된 아이템이 한도를 초과하는 동안 카운터 강조색 표시,
  '봇짐 매기'는 disabled(opacity 0.4). 한도 이하가 되면 단계색으로 활성화.
- 설정은 사운드 토글 + 전체 초기화만 (암호 변경 제외 — DESIGN.md §0-3).
- 사운드 기본 OFF, 무료 CC0 8-bit 음원, `sound/` 파일 교체만으로 변경 가능.
- 모바일 우선(시안이 모바일 세로형), 375/768/1280 검증. 데스크톱은 중앙 정렬 확장.
- 시안 카피는 글자 그대로 사용(DESIGN.md §8). 픽셀아트 에셋은 `img/` 사용, `draft/` 원본 수정 금지.
- 배포 대상: GitHub Pages (기존 min-iht.github.io/bottari 대체 여부는 push 시점에 결정).

## 기능 요구사항 (Functional Requirements)

| ID | 기능 | 요약 |
|---|---|---|
| FR-01 | 인트로 | 랜딩(GO!) → 내레이션 2장 → 이름 입력(공백 불가, 12자 이내) → 이름 치환 내레이션 |
| FR-02 | 보자기 선택 | 3단계 TierCard. 현재 단계 이상 선택 가능(상위 단계 직행 허용), 지난 단계는 비활성(grayscale) 표시 |
| FR-03 | 아이템 담기 | 사진/글/지도 추가(모달), 한도 검사, 보자기 위 카드 배치, 삭제(-) |
| FR-04 | 봇짐 매기 | 한도 조건 충족 시 활성화. 1·2단계 → 전환 연출 후 다음 단계로 아이템 이월. 3단계 → 엔딩 |
| FR-05 | 재선택 | 이월 아이템 로드, 카운터 초과 강조, 한도 이하까지 삭제 강제 |
| FR-06 | 엔딩 | 완료 화면 → 엔딩 영상 재생(muted, 종료 시 자동 전환) → 맡겨두기 분기 |
| FR-07 | 암호 공유 | 암호 설정(4자+, 확인 일치) → 암호화 URL 생성 → 복사/Web Share → 처음으로 |
| FR-08 | 수신자 열람 | `?data=` 감지 → 암호 입력 → 복호화 성공 시 열람 화면, 실패 시 에러 메시지 |
| FR-09 | 보관함 등록 | 완성 보따리를 이름과 함께 공개 저장(암호 없음) |
| FR-10 | 보관함 열람 | 그리드 목록(이름표 — 이름 길이만큼 가로 확장, 셀 폭 초과 시 `..` 말줄임) → 클릭 시 열람 화면. 빈 저장소면 빈 셀 그리드 표시 |
| FR-11 | 열람 화면 | 보자기 위 아이템 썸네일 → 클릭 시 하단 확대(사진 원본 표시, 글 전문, 지도 링크 열기) |
| FR-12 | 설정 | ⚙ 모달: 사운드 토글(상태 유지), 초기화(확인 모달 → localStorage 삭제 → 랜딩) |
| FR-13 | 사운드 | BGM 루프 + SFX 4종(클릭/담기/삭제/매듭). 토글 ON일 때만 재생 |
| FR-14 | 상태 유지 | 진행 중 이탈 시 localStorage에서 복원(이름·단계·아이템·설정) |
| FR-15 | 보관함에서 꺼내기 | 내가 등록한 보따리의 열람 화면(SCR-15)에서만 꺼내기 버튼 노출 → 확인 모달 → 삭제. 소유 증명은 등록 시 브라우저에 저장한 토큰 |

## 수용 기준 (Acceptance Criteria)

1. 인트로에서 입력한 이름이 이후 모든 화면(내레이션·말풍선·열람 제목)에 은/는 조사까지
   올바르게 치환되어 나타난다.
2. 1단계에 10개를 담아 봇짐을 매면, 2단계에 10개가 그대로 이월되고 카운터가
   `담은 개수 10 / 5`(10은 강조색)로 표시되며, 5개 이하로 덜어내기 전까지
   '봇짐 매기'가 눌리지 않는다(불투명도로 구분 가능). 3단계도 동일(한도 3).
3. 3단계 매듭 후 완료 화면·엔딩 영상을 거쳐 맡겨두기 분기에 도달한다.
4. 공유 경로: 암호(4자 이상, 확인 일치) 설정 → 생성된 URL이 2,000자 이내 →
   수신자가 올바른 암호로 열람 성공, 틀린 암호로는 내용 접근 불가.
5. 보관함 경로: 등록한 보따리가 보관함 그리드에 이름표와 함께 나타나고,
   클릭하면 아이템을 확대 열람할 수 있다. Supabase 미설정 상태에서는
   내가 등록한 보따리가 localStorage 기반으로 동작한다.
6. `js/config.js`에 Supabase 키 2개를 넣는 것만으로(코드 수정 없이) 보관함이
   공용 DB 모드로 전환된다. 설정 절차가 `docs/DB_SETUP.md`에 문서화되어 있다.
7. 설정에서 사운드 토글과 전체 초기화(확인 모달 포함)가 동작하며,
   초기화 시 랜딩으로 돌아가고 저장 데이터가 모두 삭제된다.
8. 시안 15개 화면이 DESIGN.md의 토큰·컴포넌트·카피 원문대로 구현되고,
   375/768/1280에서 가로 스크롤·요소 겹침 없이, 콘솔 에러 0건으로 동작한다.

## 데이터 모델 (Ontology)

### BottariBundle (진행 상태 — localStorage)

| 필드 | 타입 | 설명 |
|---|---|---|
| name | string | 나그네 이름 (12자 이내) |
| tier | number | 현재 단계 1/2/3 (한도 10/5/3) |
| items | array | 담긴 아이템 목록 (단계 이월 시 그대로 유지) |
| items[].type | string | "image" \| "text" \| "map" |
| items[].imageData | string | (image) 압축된 dataURL |
| items[].textContent | string | (text) 자유 텍스트 |
| items[].mapLink / mapCaption | string | (map) URL + 캡션 |
| isFinalTier | boolean | 3단계 도달 여부 (엔딩 개방) |
| settings.soundEnabled | boolean | 사운드 토글 상태 (기본 false) |

### ArchiveEntry (보관함 — Supabase `bottari_archive` 테이블 / localStorage 폴백)

| 필드 | 타입 | 설명 |
|---|---|---|
| id | uuid | 자동 생성 |
| name | string | 나그네 이름 (이름표 표시용) |
| items | json | BottariBundle.items와 동일 구조 (암호화 없음 — 공개 전제) |
| created_at | timestamp | 등록 시각 (목록 정렬 기준, 최신순) |

- 공유 URL 페이로드는 별도 저장하지 않는다(암호화된 `?data=` 자체가 저장소).
- 시드 보따리는 사용하지 않는다 — 보관함이 비어 있으면 빈 셀 그리드를 표시
  (시안 14-2-1의 6개 이름은 표기 예시일 뿐).

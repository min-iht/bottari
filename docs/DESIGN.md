Purpose:
AI(Claude Code 등)가 모든 화면을 시안과 동일한 규칙으로 구현할 수 있도록 디자인 언어를 정의한다.
v1.0(`D:\claude_code\vibe\Project\docs\DESIGN.md`)의 원칙을 계승하되, 비주얼은 시안의 **픽셀아트 레트로** 방향으로 전면 교체한다.

# 0. 확정 사항 (2026-07-16 협의)

이 문서와 이후 모든 문서·구현이 따라야 하는 결정 사항.

1. **보관함 저장소**: Supabase(무료 티어, 순수 `fetch()` REST 호출 — 노빌드 유지).
   `js/config.js`의 `SUPABASE_URL` / `SUPABASE_ANON_KEY` 두 값이 비어 있으면
   보관함은 자동으로 **localStorage 모드**(내 브라우저에만 저장, 시드 데이터 없음)로 동작한다.
   설정 절차는 `docs/DB_SETUP.md`와 `config.js` 주석에 문서화한다. *(추후 사용자가 직접 설정)*
2. **사진/글/지도 첨부 방식**: 기존 배포본(https://min-iht.github.io/bottari/)과 동일.
   - 사진: `<input type="file" accept="image/*">` → canvas 단계적 압축(JPEG, 320→48px 품질 자동 하향)
   - 글: 모달 + textarea (placeholder "소중한 순간을 글로 남겨보세요")
   - 지도: 모달 + 장소 링크(URL) + 캡션 (placeholder "이 장소에 대한 짧은 기억")
   - 아이콘도 기존 배포본 것을 그대로 사용: 🖼(사진) ✎(글) 📍(지도) ⚙(설정)
   - 공유 링크: 암호 → PBKDF2+AES-GCM 암호화 → `?data=` URL (전체 2,000자 제한, 초과 시 이미지 추가 축소)
3. **설정(⚙) 모달**: 우상단 배치는 기존 배포본과 동일하되, 항목은
   **사운드 토글 + 초기화(확인 모달 → 전체 데이터 삭제 → 처음 화면)** 2개만. '암호 변경'은 제외.
4. **단계 간 재선택 규칙**: 이전 단계 아이템이 그대로 로드된다.
   카운터 `담은 개수 10 / 5`에서 현재 개수가 제한을 초과하는 동안 **강조색**(§4.3)으로 표시,
   제한 이하가 되면 검은색으로 복귀. 제한 이하가 되기 전까지 '봇짐 매기'는
   **클릭 불가 + 불투명도 40%**, 충족 시 기본 차콜 버튼으로 활성화되고
   터치(pressed) 순간에만 단계색(§4.2)으로 변한다. 1단계의 0개 상태에도 동일 적용.
5. **사운드**: 무료 8-bit 음원 사용(§9). 파일 교체만으로 바꿀 수 있는 구조로.
6. **전체 배경색 = `#FFFFFF` 고정** (2026-07-16 사용자 확정): 모바일 화면 배경과
   데스크톱에서 남는 좌우 여백까지 전부 흰색. `COLOR-BG`를 `#FFFFFF`로 갱신(§4.1).

# 1. Design Principle

## 1.1 Core Philosophy

v1.0과 동일 — 보따리는 "기억을 담는 서비스"가 아니다.
**선별(Curation) → 비움(Release)** 이라는 감정의 흐름을 경험하는 인터랙션이다.
사용자는 무엇을 더 담는 사람이 아니라, 무엇을 남길 것인가를 결정하는 사람이다.

여기에 v2.0의 형식이 더해진다:
**레트로 픽셀 게임의 문법**(도트 캐릭터, 대화 상자, 픽셀 폰트)으로
무거운 주제(디지털 기억 유산)를 가볍고 다정하게 감싼다.

## 1.2 Keywords

Quiet · Warm · Pixel · Winter(눈) · Pastel · Travel · Narrative First

## 1.3 Design Rules

DO
- 여백을 적극적으로 사용한다. 화면당 정보는 한 덩어리만.
- 감정은 캐릭터와 대사(대화 상자)로 전달한다.
- 픽셀 엣지를 유지한다 — 도트 에셋은 `image-rendering: pixelated`로 확대.
- 파스텔 3색(핑크/노랑/하늘)은 보자기·캐릭터·포인트에만, UI 뉴트럴은 흑백 계열 유지.

DON'T
- 안티앨리어싱된 부드러운 일러스트 (픽셀 세계관 붕괴)
- 화려한 그라데이션, 과한 그림자, 라운드 큰 모서리
- 복잡한 인터랙션, 과도한 정보
- 시안 카피 임의 수정 (§8의 문안은 글자 그대로 사용)

# 2. Layout System

- Base Grid: 8px / Spacing Scale: 4, 8, 12, 16, 24, 32, 48, 64, 96
- 시안 원본 비율: 1320×2868 (모바일 세로형)
- Container: Mobile 100%, **Max Width 480px**, Desktop은 중앙 정렬(레이아웃 동일, 좌우 여백만 증가)
- Safe Area: Top 24 / Bottom 32
- Screen Template: `Header(제목+⚙) → Content → Bottom Action → Modal(Optional)`
  - 내레이션형 화면(인트로/전환/완료)은 Header 없이 Content 중앙 정렬
- 반응형 검증 폭: 375 / 768 / 1280 (가로 스크롤 0, 요소 겹침 0)

Rules
- 카운터(`담은 개수 n / limit`)는 `TYPO-Strong` 사용(버튼·대제목과 동일 계열).
- 픽셀 폰트는 홀수 px 크기 금지(블러 방지). 기준 크기: Display 40~48 / 제목 28 / 본문 17~18 / 캡션 13~14 / 버튼 20
- 본문 내레이션은 중앙 정렬 허용(시안 기준). 긴 문장은 최대 3줄.
- letter-spacing은 기본 0, 픽셀 폰트 특성상 자간 보정 금지.

# 5. Border · Radius · Shadow

- **픽셀 보더**: 카드/대화 상자/입력 박스는 `2px solid COLOR-Text` + **radius 2px** (사용자 확정).
  시안의 계단식 모서리 재현이 필요한 요소(대화 상자)는 border-image 또는 box-shadow 스텝 기법 사용 가능.
- Shadow: 사용하지 않는다(픽셀 세계관). 모달 분리는 오버레이 어둡기로만.
- **버튼 보더 radius: 2px** (사용자 확정).

# 8. Copy (원문 고정)

시안의 문안은 **글자 그대로** 사용한다. 수정 금지. 이름 자리는 `{이름}`, 조사는 은/는 자동 선택.

- 태그라인: `디지털 기억 유산`
- SCR-01: `보따리 싸러 가볼까?` / `GO!`

# 9. Motion & Sound

Motion
- Duration: Fast 150ms / Normal 250ms / Slow 400ms. Opacity·TranslateY·Scale만. Bounce/Elastic 금지.
- 화면 전환: 페이드(Normal). 내레이션 텍스트: 페이드 인.
- 눈 내림: `COLOR-Snow` 픽셀 눈송이 3~4개가 배경에서 천천히 하강(CSS keyframes, 캐릭터 등장 화면 한정).
- 봇짐 매기: 펼친 보자기 → 묶인 보따리 이미지 스왑 + Scale 0.98→1 (Slow).
- 아이템 담기/삭제: 카드 페이드 + TranslateY 4px.
- SCR-09: `video_ending.mp4` 재생, 종료(ended) 시 SCR-10으로 페이드.

Sound (8-bit, 무료 CC0)
- 소스: Pixabay / OpenGameArt의 CC0 8-bit 음원에서 선택. *(확인 필요: 최종 음원은 구현 단계에서 후보 제시)*
- 구성: BGM 루프 1개(잔잔한 칩튠) + SFX 4개(클릭 / 담기 / 삭제 / 봇짐 매기·완료).
- 파일 위치 — BGM은 `bgm/bottari_bgm.mp3`, SFX는 `sound/`에 `sfx_click.mp3`, `sfx_add.mp3`, `sfx_remove.mp3`, `sfx_tie.mp3`. 교체 시 파일만 갈아끼우면 되도록 이름 고정.
- 음원은 -14 LUFS(스트리밍 표준)로 정규화해서 넣는다(재생 측 볼륨 보정 없음).
- **기본 ON.** 단, autoplay 정책상 부팅 즉시 재생은 불가 — 첫 클릭/터치에서 시작. 토글 OFF 시 즉시 정지. 상태는 localStorage 유지.
